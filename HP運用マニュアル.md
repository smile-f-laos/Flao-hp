# Smile F LAOS HP運用マニュアル

このマニュアルは、Smile F LAOS公式ホームページを後輩が安全に更新・管理できるようにするためのものです。

`README.md` はリポジトリ全体の簡単な説明用、  
この `HP運用マニュアル.md` は実際に作業するときの詳しい説明書として使います。

---

## 0. はじめに

このホームページは、Smile F LAOSの活動内容、ラオスコーヒー、教育支援、スタディツアー、協力団体、お知らせなどを外部に伝えるための公式ホームページです。

更新するときは、ただ情報を書き換えるだけでなく、以下の点を大切にしてください。

- 情報が正確であること
- 見る人にとってわかりやすいこと
- 協力してくださっている方々への敬意があること
- Smile F LAOSらしい温かさが伝わること
- スマホでも見やすいこと

---

## 1. このマニュアルの目的

このマニュアルの目的は、後輩が以下の作業をできるようにすることです。

- GitHub上でHTMLファイルを編集する
- ホームページの文章を更新する
- 画像を追加・差し替えする
- お知らせを追加する
- 各ページの役割を理解する
- 更新後に正しく表示されているか確認する
- ホームページを壊さないように安全に作業する

GitHubのすべての機能を覚える必要はありません。  
このマニュアルでは、Smile F LAOSのホームページを更新するうえで必要な操作だけを説明します。

---

## 2. HP更新の全体像

ホームページを更新するときの基本的な流れは以下の通りです。

1. GitHubにログインする
2. Smile F LAOSのホームページ用リポジトリを開く
3. 編集したいHTMLファイルを開く
4. 右上の鉛筆マークを押して編集する
5. 必要な文章・画像・リンクを変更する
6. `Preview` で確認する
7. commit messageを書く
8. `Commit changes` を押す
9. 公開されたホームページを確認する
10. PC表示とスマホ表示の両方を確認する

---

## 3. GitHubの基本操作

### 3-1. GitHubにログインする

GitHubにアクセスし、Smile F LAOSのホームページを管理しているアカウントでログインします。

個人アカウントで作業する場合と、Organizationで作業する場合があります。  
どちらで管理しているかは、引き継ぎ時に必ず確認してください。

---

### 3-2. リポジトリを開く

リポジトリとは、ホームページのファイルが入っているフォルダのようなものです。

この中に、以下のようなファイルがあります。

- `index.html`
- `coffee.html`
- `education.html`
- `studytour.html`
- `gallery.html`
- `partners.html`
- `news.html`
- `README.md`
- `HP運用マニュアル.md`

---

### 3-3. ファイルを編集する

編集したいファイルをクリックします。

例：

トップページを編集したい場合は、`index.html` を開きます。

ファイルを開いたら、右上にある鉛筆マークを押します。  
すると、ファイルの中身を編集できるようになります。

---

### 3-4. Previewで確認する

編集したあとは、`Preview` を押して、変更内容に大きなミスがないか確認します。

ただし、HTMLファイルの場合、GitHub上のPreviewだけでは実際の見た目を完全には確認できません。  
commitしたあと、必ず公開されているホームページを開いて確認してください。

---

### 3-5. commitする

commitとは、変更内容を保存する記録のことです。

commit messageには、何を変更したのかを短く書きます。

例：

- トップページの文章を修正
- コーヒーページの商品説明を更新
- お知らせページにイベント情報を追加
- スタディツアーページの写真を差し替え
- ヘッダーのリンクを修正

---

### 3-6. 公開されたHPを確認する

commitしたあと、実際のホームページを開いて確認します。

確認する内容は以下です。

- 変更内容が反映されているか
- 文章が途中で切れていないか
- 画像が表示されているか
- リンクが正しく動くか
- PC表示で崩れていないか
- スマホ表示で崩れていないか

---

## 5. 全ページ共通の編集ルール

### 5-1. headerの扱い

headerはページ上部に表示されるメニュー部分です。

現在のホームページでは、headerが固定されています。  
つまり、スクロールしても画面上部に残るようになっています。

headerを変更するときは注意が必要です。

特に以下の点に気をつけてください。

- 1ページだけheaderの内容を変えない
- ナビゲーションのリンク先を間違えない
- スマホ表示でメニューが開くか確認する
- ロゴや団体名の表記を変える場合は確認する

---

### 5-2. メインカラー

現在のホームページでは、メインカラーとして以下の色を使っています。

```text
#a44f30
```

CSS内では、以下のように設定されています。

```css
:root {
  --primary: #a44f30;
}
```

色を変更すると、ホームページ全体の印象が大きく変わります。  
基本的に、勝手に変更しないでください。

---

### 5-3. スマホ表示の確認

ホームページはPCだけでなく、スマホで見る人も多いです。

更新後は必ずスマホ表示を確認してください。

確認するポイント：

- ヘッダーが崩れていないか
- メニューが開くか
- 文字が小さすぎないか
- 画像がはみ出していないか
- ボタンが押しやすいか
- 横スクロールが発生していないか

---

### 5-4. リンクの確認

リンクを追加・変更した場合は、必ず実際にクリックして確認してください。

例：

```html
<a href="coffee.html">コーヒー販売を見る</a>
```

この場合、`coffee.html` が実際に存在している必要があります。

よくあるミス：

- ファイル名のスペルミス
- `.html` のつけ忘れ
- `studytour.html` と `study-tour.html` の混同
- 大文字と小文字の違い
- 存在しないページへのリンク

---

### 5-5. 文章の書き方

文章を更新するときは、以下を意識してください。

- 初めて見る人にもわかる言葉にする
- 内輪だけに伝わる表現を避ける
- 誤字脱字を確認する
- 企業名・団体名・人名を間違えない
- 活動内容を誇張しすぎない
- 現地の方への敬意が伝わる表現にする

### head部分

`head` には、ページの基本設定とCSSが書かれています。

例：

```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Smile F LAOS</title>
  <style>
    :root {
      --primary: #a44f30;
    }
  </style>
</head>
```

基本的に、`head` 内はむやみに変更しないでください。

特に以下は消さないようにしてください。

```html
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

これを消すと、日本語が文字化けしたり、スマホ表示が崩れたりする可能性があります。

---

### CSS部分

現在の `index.html` は、HTMLファイル内にCSSが書かれています。

CSSとは、色・余白・文字サイズ・配置など、見た目を決める部分です。

例：

```css
:root {
  --primary: #a44f30;
  --bg: #ffffff;
  --text-main: #000;
}
```

この中の `--primary` がメインカラーです。

```css
--primary: #a44f30;
```

この色は、headerやボタンなどに使われています。  
色を変更すると全体の雰囲気が変わるため、基本的には変更しないでください。

---

### hero画像を差し替える方法

画像を差し替える場合は、以下の流れで作業します。

1. 新しい画像をリポジトリにアップロードする
2. 画像ファイル名を英数字にする
3. `index.html` のhero部分を探す
4. `img src="..."` の中身を新しい画像ファイル名に変更する
5. commitする
6. 公開後に画像が表示されるか確認する

例：

変更前：

```html
<img src="HP1.jpg.jpg" alt="Smile F LAOSの活動写真1" class="slide active">
```

変更後：

```html
<img src="laos-school-2026.jpg" alt="ラオスの小学校での活動写真" class="slide active">
```

---

### hero画像を増やす方法

画像を増やしたい場合は、`hero-bg` の中に `img` タグを追加します。

例：

```html
<img src="new-hero-photo.jpg" alt="Smile F LAOSの活動写真" class="slide">
```

注意点：

- 追加する画像には基本的に `active` をつけない
- `active` は最初に表示する1枚だけにつける
- 画像ファイル名とHTMLの記述を完全に一致させる
- 写真の使用許可を確認する

3. ラオスで得た学びをもとにニーズに合った支援を探求し続ける

### スマホメニュー用JavaScript

スマホでメニューを開くためのコードです。

```javascript
const header = document.querySelector(".site-header");

navToggle.addEventListener("click", () => {
  header.classList.toggle("is-open");
});
```

このコードは、スマホ表示でメニューを開閉するために必要です。  
意味がわからない場合は、変更しないでください。

---

### スライドショー用JavaScript

hero画像を自動で切り替えるためのコードです。

現在は、約5秒ごとに画像が切り替わるようになっています。

例：

```javascript
const slides = document.querySelectorAll(".slide");
let currentSlide = 0;

setInterval(() => {
  slides[currentSlide].classList.remove("active");
  currentSlide = (currentSlide + 1) % slides.length;
  slides[currentSlide].classList.add("active");
}, 5000);
```

`5000` は5秒を表しています。

変更例：

```javascript
}, 3000);
```

にすると3秒ごとに切り替わります。

ただし、切り替わりが早すぎると見づらくなるため、基本的には `5000` のままで問題ありません。

## 9. よくあるミスと対処法

### 9-1. HPに変更が反映されない

原因として考えられること：

- commitできていない
- 反映に時間がかかっている
- 違うファイルを編集している
- ブラウザのキャッシュが残っている

対処法：

- GitHubでcommit履歴を確認する
- 数分待ってから再読み込みする
- Ctrl + F5 で強制更新する
- 編集したファイルが正しいか確認する

---

### 9-2. 画像が表示されない

原因として考えられること：

- 画像ファイル名が違う
- 拡張子が違う
- 画像がアップロードされていない
- 大文字・小文字が違う
- ファイル名に空白がある

対処法：

- 画像ファイル名をコピーしてHTMLに貼る
- `.jpg`、`.jpeg`、`.png` を確認する
- GitHub上に画像があるか確認する

---

### 9-3. レイアウトが崩れた

原因として考えられること：

- 閉じタグを消した
- CSSを間違えて編集した
- `class` 名を変えた
- 必要な `div` を消した

対処法：

- 直前に編集した場所を確認する
- 閉じタグを確認する
- 前のcommitとの差分を見る
- わからなければ詳しい人に確認する

---

### 9-4. リンクが開かない

原因として考えられること：

- ファイル名が違う
- `.html` をつけ忘れている
- 存在しないページにリンクしている
- `href` の書き方が間違っている

正しい例：

```html
<a href="coffee.html">コーヒーページへ</a>
```

---

### 9-5. スマホで表示が崩れる

原因として考えられること：

- 画像が大きすぎる
- 横幅を固定しすぎている
- スマホ用CSSを消した
- headerの構造を変えた

対処法：

- スマホで実際に開いて確認する
- 画像サイズを小さくする
- `@media` から始まるCSSをむやみに消さない
