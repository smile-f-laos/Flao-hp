# Smile F LAOS ホームページ運用マニュアル

このリポジトリは、Smile F LAOSの公式ホームページを管理するためのものです。  
後輩が引き継いだあとも、ホームページを安全に更新できるように、このREADMEに基本的な使い方をまとめています。

---

## 1. このホームページの目的

このホームページは、Smile F LAOSの活動内容、ラオスコーヒー、スタディツアー、協力団体、お知らせなどを外部に伝えるための公式ホームページです。

単に情報を載せるだけでなく、エフラオが大切にしている

- ラオスのフェアトレードコーヒーの魅力を広めること
- 学生ならではの知識を活かして、ラオスの子どもたちの自立を促す教育支援を行うこと
- ラオスで得た学びをもとに、ニーズに合った支援を探求し続けること

を、見に来てくれた人にわかりやすく伝えることを目的としています。

---

## 2. 基本的な更新の流れ

ホームページを更新するときは、基本的に以下の流れで作業します。

1. GitHubにログインする
2. このリポジトリを開く
3. 編集したいファイルを開く
4. 右上の鉛筆マークから編集する
5. 内容を変更する
6. `Preview` で確認する
7. commit messageを書く
8. `Commit changes` を押す
9. 公開されたホームページを確認する
10. スマホ表示でも崩れていないか確認する

---

## 3. よく使うGitHub用語

### GitHub

ホームページのコードや画像を保存・管理する場所です。  
誰が、いつ、どこを変更したかを記録できます。

### リポジトリ

ホームページのデータが入っているフォルダのようなものです。  
HTMLファイル、画像ファイル、READMEなどがここに入っています。

### commit

変更内容を保存する記録のことです。  
「何を変更したか」があとから見返せるようになります。

commit messageには、何を変更したのかを短く書きます。

例：

- トップページの文章を修正
- コーヒーページの商品説明を更新
- お知らせページにイベント情報を追加
- スタディツアーページの写真を差し替え

### branch

ホームページを直接壊さないための作業用コピーのようなものです。  
小さな文章修正であればそのまま編集しても大丈夫ですが、大きなデザイン変更やページ追加をするときは、branchを作って作業する方が安全です。

### Pull Request

変更内容を本番に反映する前に、他の人に確認してもらうための機能です。  
大きな変更をするときは、Pull Requestを作って確認してもらうようにしてください。

## 5. 各ページの役割

### `index.html`：トップページ

ホームページに来た人が最初に見るページです。  
Smile F LAOSの第一印象を決めるページなので、文章や画像を変更するときは慎重に行ってください。

主に載せる内容：

- 団体の概要
- エフラオらしさが伝わるメッセージ
- 各ページへの導線
- メインビジュアルや活動写真

注意点：

- キャッチコピーを変更する場合は、代表や広報担当に確認する
- 画像を変更したら、PCとスマホの両方で表示を確認する
- 他ページへのリンクが切れていないか確認する

---

### `coffee.html`：コーヒーページ

エフラオが販売しているラオスコーヒーの魅力を伝えるページです。

主に載せる内容：

- ラオスコーヒーについて
- フェアトレードについて
- 商品の特徴
- 中煎り・深煎りの説明
- 販売している商品の情報

注意点：

- 価格、内容量、商品名を変更するときは必ず確認する
- 焙煎度や味の説明を適当に変更しない
- 商品写真を変更した場合は、画像が正しく表示されているか確認する
- イベント販売の情報と矛盾がないようにする

---

### `study-tour.html`：スタディツアーページ

ラオスでの現地活動を伝えるページです。  
エフラオの支援内容や、現地での学びを伝える重要なページです。

主に載せる内容：

- スタディツアーの目的
- 現地での活動内容
- 小学校での教育支援
- コーヒー農家さんへの聞き取り
- 現地で得た学び
- 活動写真

注意点：

- 子どもや現地の方の顔がはっきり写っている写真を使う場合は、使用してよい写真か確認する
- 活動内容を誇張しすぎない
- 年度ごとの実績を更新するときは、人数や訪問先の情報を正確に書く
- 現地の方への敬意が伝わる表現にする

---

### `partners.html`：協力団体ページ

Smile F LAOSを支えてくださっている企業・団体を紹介するページです。

主に載せる内容：

- 協力団体の正式名称
- どのような形で協力していただいているか
- 企業・団体との関係性
- 協力内容の説明

注意点：

- 企業名・団体名を絶対に間違えない
- 敬称や表現に注意する
- 協力内容を変更する場合は、事実確認を行う
- ロゴや写真を使用する場合は、使用してよいものか確認する

---

### `news.html`：お知らせページ

イベント情報や活動報告を掲載するページです。

主に載せる内容：

- 販売会のお知らせ
- イベント出展情報
- 活動報告
- 新歓情報
- その他、外部に知らせたい情報

注意点：

- 日付、時間、場所を間違えない
- 終了したイベントの情報をどう扱うか確認する
- 外部リンクを貼る場合は、リンク先が正しいか確認する
- 公開前に必ず内容を見直す

## 6. 画像を追加・変更するときの注意

画像を追加するときは、以下の点に注意してください。

- ファイル名は英数字にする
- 日本語のファイル名はできるだけ使わない
- 画像サイズが大きすぎる場合は、圧縮してから使う
- 画像を差し替えたら、HTML側のファイル名も正しく変更する
- PCとスマホの両方で画像が崩れていないか確認する

画像ファイル名の例：

```text
coffee-medium-roast.jpg
coffee-dark-roast.jpg
study-tour-2025.jpg
laos-school-workshop.jpg
partners-atj.jpg
```

## 7. デザインの共通ルール

現在のホームページでは、エフラオらしい落ち着いた茶色をメインカラーとして使用しています。

メインカラー：

```text
#a44f30
```

注意点：

- 色を変更する場合は、全ページの統一感が崩れないようにする
- headerは全ページで共通のデザインにする
- 1ページだけナビゲーションの内容が違う状態にしない
- 文字が読みにくくなる色の組み合わせは避ける

## 8. 更新するときにやってはいけないこと

以下の作業は、ホームページが壊れる原因になるため注意してください。

- 意味が分からないコードをまとめて消す
- 画像ファイル名を変えたのに、HTML側を修正しない
- headerやナビゲーションを1ページだけ変更する
- 確認せずに企業名・団体名・人名を変更する
- スマホ表示を確認せずに公開する
- 個人情報を不用意に載せる
- 顔写真の使用可否を確認せずに掲載する
- 大きなデザイン変更を相談なしで行う

## 11. 困ったときの対応

### 変更したのにホームページに反映されない

少し時間を置いてから、もう一度ページを更新してください。  
それでも反映されない場合は、commitが完了しているか確認してください。

### 画像が表示されない

以下を確認してください。

- 画像ファイルがリポジトリ内にあるか
- HTMLに書いたファイル名と画像ファイル名が一致しているか
- 拡張子が `.jpg`、`.png` など正しく書かれているか
- 大文字・小文字が一致しているか

例：

```html
<img src="coffee.jpg" alt="ラオスコーヒー">
```

この場合、画像ファイル名も完全に `coffee.jpg` である必要があります。

### レイアウトが崩れた

直前に変更した部分を確認してください。  
特に、以下の閉じタグを消していないか確認してください。

```html
</div>
</section>
</main>
</body>
</html>
```
